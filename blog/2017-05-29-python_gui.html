Learning Python UI programming

Posted on 29th May 2017

Another new task: get going with some GUI programming!

Some references

  • Tkinter 8.5 reference: a GUI for Python - Lots and lots. On the web, or a 168 page PDF file.
  • TkDocs - Cross platform, but lots of Python examples. I found this pretty good!
  • Official docs
  • Effbot Book - A "work in progress" and old, but still, so I've found, is an excellent reference and guide.

Hints and tip

  • Tooltips in tkinter
  • Basic quick start guide - Rather basic, but a nice intro.
  • Brief intro - Nice one-page introduction (with lots of images; handy when discussing GUIs!)
  • OOP in Python chapter - More of an introduction than anything else.
  • Threads and Tkinter - How to send data back to the GUI thread.
  • Modal dialogs in Tkinter - The method .wait_window() is necessary but not sufficient.

Some books

As culled from Leeds University Library:

  • User interface design : a software engineering perspective 2005. Sounds interesting though.
  • Designing the user interface : strategies for effective human-computer interaction 2010. Seems like a bit more of a textbook. Having read, it does indeed seem to be a textbook, and a somewhat confused one I found. Each chapter seemed rather self-contained, without much linking them, and without much order. This is a fast-changing area of technology (think: mobile phones, or single page web apps) and the book felt somewhat out of date, with the odd update bolted on. Unfortunately, it did little of what I was hoping for: practical ideas, from the development side, of how to design and write GUI code.
  • Usability engineering 2000. Picked out after looking at books on the shelf.
